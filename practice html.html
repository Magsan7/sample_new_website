HTML — Complete Notes

Concise + practical notes in bullet style. Definitions • Syntax • Examples • Best practices • Interview Qs

1. Introduction & History of HTML

  • HTML = HYPERTEXT MARKUP LANGUAGE — describes structure of web pages.
  • Invented by Tim Berners-Lee (1991) to publish and link documents on the Web.
  • HTML uses tags (e.g., <h1>, <p>) to mark elements.
  • HTML5 added semantic tags, multimedia, canvas, offline APIs, and modern web capabilities.
  • Important: HTML = structure (use CSS for style, JS for behavior).

2. Structure of an HTML Document

  • Every page typically contains: <!DOCTYPE html>, <html>, <head>, <body>.
  • <head> holds metadata: <title>, <meta>, links to CSS and scripts.
  • <body> contains visible content: headings, paragraphs, images, forms, etc.

Basic template

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Page Title</title>
  </head>
  <body>
    <h1>Hello</h1>
    <p>Content here.</p>
  </body>
</html>

3. Elements, Tags, and Attributes

  • Element = start tag + content + end tag. Example: <p>text</p>.
  • Tag = markup like <a>, <img>. Self-closing tags: <br>, <img>.
  • Attribute adds information inside a tag: <img src="img.jpg" alt="desc">. Use lowercase and quotes.
  • Best practice: Always include alt for images, and meaningful attributes (id, class, title, aria-*).

Common attributes

  • id — unique identifier.
  • class — reusable style/selector.
  • title — mouse tooltip / accessibility hint.
  • data-* — custom data attributes for JS.

4. Headings, Paragraphs, and Lists

  • Headings: <h1> (top) to <h6> (small). Use for document structure (semantic).
  • Paragraph: <p> — block text with margin. Use for paragraphs, not for spacing.
  • Lists:
    • <ul> — unordered (bullets)
    • <ol> — ordered (numbers)
    • List item: <li>. Use lists for actual lists, not layout.

Example

<h2>Shopping</h2>
<p>Buy these items:</p>
<ul>
  <li>Milk</li>
  <li>Bread</li>
</ul>

5. Links, Images, Tables, and Forms

Links

  • Anchor: <a href="URL">link text</a>. Use absolute or relative URLs.
  • Use target="_blank" to open new tab and rel="noopener noreferrer" for security.

Images

  • <img src="path.jpg" alt="description" width="300">. Always provide descriptive alt.
  • Use responsive images: srcset, sizes, and picture element.

Tables

  • Basic tags: <table>, <tr>, <th>, <td>.
  • Use <caption> and scope on headers for accessibility.

Forms

  • Form basics: <form action="/submit" method="post">.
  • Input types: text, password, email, number, checkbox, radio, file, date.
  • Use label for="id" to link labels and inputs — important for accessibility.

Form example

<form action="/signup" method="post">
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  <button type="submit">Submit</button>
</form>

6. Semantic Tags, Multimedia, and Iframes

  • Semantic elements: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>.
  • Use semantic tags for accessibility, SEO, and clearer structure.
  • Audio/Video: <audio controls>, <video controls width="640">. Provide captions via <track>.
  • Iframes: <iframe src="..." title="desc"></iframe>. Use title and sandbox attributes for security.

Multimedia example

<video controls>
  <source src="video.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

7. HTML5 Features & APIs

  • Canvas API: draw 2D graphics with JavaScript (canvas.getContext('2d')).
  • Audio/Video APIs: programmatic control of playback, events and tracks.
  • LocalStorage / SessionStorage: store key/value data in the browser (persistent or per-session).
  • Geolocation API: request user's location (permission required).
  • Drag & Drop, Web Workers, Service Workers: background tasks and offline web apps.

Canvas quick example

<canvas id="c" width="200" height="100"></canvas>
<script>
const c=document.getElementById('c'),ctx=c.getContext('2d');
ctx.fillStyle='lightblue'; ctx.fillRect(0,0,200,100);
ctx.fillStyle='black'; ctx.font='16px sans-serif'; ctx.fillText('Canvas demo',10,50);
</script>

8. Best Practices & Common Mistakes

  • Structure first: write semantic HTML before styling with CSS.
  • Accessibility: use alt, label, role, and ARIA when needed.
  • Responsive images: use srcset and picture.
  • Never use tables for layout — tables are for tabular data only.
  • Validate HTML using the W3C Validator to catch errors early.
  • Security: when embedding third-party content, use rel="noopener" and sandbox iframes.

9. Interview Questions & Key Takeaways

Quick Interview Questions

  • Q: Difference between <div> and <span>? — A: <div> is block-level, <span> is inline.
  • Q: What is semantic HTML? — A: Tags that convey meaning: <article>, <nav>, etc.
  • Q: How to make images responsive? — A: Use CSS max-width:100% or srcset and sizes.
  • Q: When to use localStorage vs cookies? — A: localStorage for client-side persistent storage (bigger, not sent to server); cookies for server-read small data.

Key Takeaways

  • HTML = structure; separate concerns: CSS for presentation, JS for behavior.
  • Prefer semantic tags for accessibility and SEO.
  • Progressive enhancement: ensure basic functionality without JS; then enhance.
  • Always consider accessibility (keyboard nav, screen readers, labels).
